Create a new mart table for: $ARGUMENTS

Format: `/new-mart dim customers` or `/new-mart fact order_items`

This project uses a star schema in the `mart` schema. Read the relevant stg table from `02_data_cleaning_transformation/` and the model documentation from `03_data_modeling/model_documentation.md` before writing SQL.

---

**For dimension tables** (`dim_`):

Follow the pattern in `03_data_modeling/01_dimension_tables/dim_customer.sql`:

- Use `CREATE TABLE mart.dim_X (...)` with explicit column definitions and data types
- First column: `X_key INT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY`
- Second column: `X_id TEXT UNIQUE` (the business key from stg)
- Include all descriptive attributes from the stg table
- Carry forward relevant quality flags from stg (missing_X_flag, etc.)
- Add `is_unknown_X BOOLEAN` as the last flag column
- Add `created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP`
- Always insert an unknown fallback row with a forced key of -1:
  `INSERT INTO mart.dim_X (...) OVERRIDING SYSTEM VALUE VALUES (-1, 'UNKNOWN', ...)`
- Then `INSERT INTO mart.dim_X SELECT ... FROM stg.stg_X WHERE ghost/duplicate flags = FALSE`

**For fact tables** (`fact_`):

Follow the pattern in `03_data_modeling/02_fact_tables/fact_order_items.sql`:

- Use `CREATE TABLE mart.fact_X (...)` with explicit column definitions
- Foreign keys reference dimension surrogate keys (customer_key, product_key, etc.)
- Use `COALESCE(dim.X_key, -1)` to map unmatched records to the unknown member
- Include degenerate dimension IDs (original business IDs kept for reference)
- Include all measures (amounts, quantities, flags)
- Add `created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP`
- No unknown fallback row needed in fact tables

---

Save the file in the correct subfolder:
- `03_data_modeling/01_dimension_tables/dim_X.sql`
- `03_data_modeling/02_fact_tables/fact_X.sql`
